# DreamChamber SAFE RECOVERY Doctrine (Pinned)

**MICKY-P is a mine, not a migration casualty.**

## The Three Passes

### Pass 1 — Discover + Copy
- Inventory only
- Copy only
- **Never delete originals**
- Emit manifests

### Pass 2 — Verify + Classify
- Hashes + counts
- Identify duplicates/conflicts
- Vendor-by-vendor plugin review

### Pass 3 — Selective Migration
- Move only what is verified
- Re-authorize plugins cleanly
- Stabilize GOD before any cleanup

## Hard Rules
- Default command is `audit`
- No plugin installs on day one
- No automation of Logic wiring
- ENGR_KEITH is opt-in after validation
- Cleanup is manual approval only

## Why
- Protect creative history
- Preserve vendor licensing integrity
- Keep GOD stable
- Make recovery reusable as data (CSV + JSONL), not a one-off story

**First pass: discover and copy.
Second pass: verify and classify.
Third pass: selectively migrate.
Only after that: clean, nuke, automate.**
