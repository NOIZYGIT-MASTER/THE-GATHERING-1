# Stage 04 — NOIZYFISH Design System

## Goal

Forge the NOIZYFISH visual identity. Deep ocean. Sacred memory. Museum-grade calm.

## Brand Essence

- archival restraint
- cinematic pacing
- oceanic depth
- elegant permanence

## Deliverables

### Typography scale

- Display sizes for heroes
- Body sizes for content
- Metadata sizes for labels

### Color tokens

- Abyss black
- Midnight blue
- Ocean cyan (primary)
- Silver oxide (accents)
- Controlled glow (earned only)

### Components

- Archive card variants (featured, compact, list)
- Archive grid behavior
- Metadata presentation patterns
- Status indicators (verified, pending, archived)
- Button variants (ocean, ghost)

### CSS/Tailwind

- `apps/noizyfish/app/globals.css` with design system classes
- Theme tokens in `packages/config/tailwind.config.js`

### Motion rules

- Subtle, calm, earned
- No flashy gradients
- No plastic glassmorphism

## Exit criteria

- NOIZYFISH feels archival, not product-marketing
- Cards do not feel ecommerce-like
- Dark theme feels deep, not flashy
- Design system is documented in CSS

## Checkpoint

After completion:

1. List design files changed
2. Confirm visual identity is distinct
3. State any design compromises
4. Confirm readiness for Stage 05
