"No voice is synthesized, licensed, or distributed without an explicit, verifiable, revocable consent manifest attached at the point of creation."
Four Primitives
The Consent Kernel is composed of exactly four primitives. Every operation in the HVS pipeline — voice model creation, Descendant synthesis, licensing, distribution, and payment — requires all four to be present and valid. No primitive can be omitted. No primitive can be overridden by a licensee.
Primitive 01
Identity Asset
The immutable record that ties a voice model to its human origin. Establishes provenance, links to the artist's verified identity, and anchors all downstream rights.
artist_id string required — Unique artist identifier (e.g. RSP_001)
legal_name string required — Full legal name of voice owner
voice_model_id string required — UUID of the registered voice model
registration_date ISO 8601 required — Date model was registered
jurisdiction string required — Governing legal jurisdiction
union_affiliations string[] optional — SAG-AFTRA, ACTRA, Equity, etc.
watermark_hash string required — SHA-256 of embedded voice DNA watermark
Primitive 02
License Policy
The rules governing how a Descendant may be used. Defines permitted categories, duration, territory, exclusivity, and the seven Never Clauses that cannot be overridden.
license_id string required — Unique license instance ID
tier enum required — community | standard | premium
permitted_uses string[] required — Allowed content categories
never_clauses string[] required — 7 creative vetoes (immutable)
territory string[] required — Geographic scope
duration_days integer required — License term in days
exclusivity boolean optional — Exclusive use grant (Premium only)
auto_renew boolean optional — Auto-renewal flag
Primitive 03
Revenue Policy
The economics engine. Enforces the 75/25 split, calculates union contribution from the artist's share based on Tracked & Stacked profile data, and ensures artist cost is always $0.
artist_pct integer required — Always 75 (artist's gross share)
platform_pct integer required — Always 25 (NOIZY.AI hosting)
union_contribution_pct number required — 2-10% from artist's 75% based on tier
artist_net_pct number required — Calculated: 75 minus union_contribution_pct
artist_upfront_cost integer required — Always 0. Artist never pays.
licensee_pays boolean required — Always true. All fees are licensee-facing.
residual_schedule string optional — 13-week cycle for broadcast tier
payment_trigger enum required — on_generation | on_license | on_distribution
Primitive 04
Revocation Policy
The kill switch. Artist can revoke consent at any time, for any reason, at zero cost to them. Revocation fee ($1,000) is charged to the licensee, not the artist. The artist never pays to say no.
revocation_available boolean required — Always true
revocation_cost_to_artist integer required — Always 0
revocation_fee_licensee integer required — $1,000 processing fee
grace_period_hours integer required — Hours after revocation before full halt
posthumous_policy enum optional — halt_all | beneficiary_decides | continue_limited
beneficiary_ids string[] optional — Designated heirs for posthumous decisions
emergency_halt boolean required — Instant suspension, no grace period

Every synthesis request passes through this validation chain before a single audio sample is generated. If any step fails, the request is blocked and a ledger entry is written recording the rejection.

Validation Rules
These rules are enforced at the Consent Kernel level. A manifest that fails any BLOCK-severity rule is rejected and the synthesis request is denied. WARN-severity rules generate alerts but do not block.
CK-V001
Never Clauses Completeness
All 7 never clauses must be present in every manifest. No partial enforcement. A manifest missing even one clause is invalid.
BLOCK
CK-V002
Identity-Model Binding
The artist_id must match a verified artist in the registry. The voice_model_id must be registered to that artist. Cross-identity synthesis is impossible.
BLOCK
CK-V003
Revenue Invariants
artist_pct must equal 75. platform_pct must equal 25. artist_upfront_cost must equal 0. licensee_pays must be true. Any deviation is schema-enforced invalid.
BLOCK
CK-V004
Union Contribution Range
union_contribution_pct must fall between 2 and 10 inclusive. Must match the tier calculated from the artist's Tracked & Stacked profile data.
BLOCK
CK-V005
Artist Net Calculation
artist_net_pct must exactly equal (75 - union_contribution_pct). Any arithmetic mismatch invalidates the manifest.
BLOCK
CK-V006
Kill Switch Availability
revocation_available must always be true. revocation_cost_to_artist must always be 0. The artist's right to say no is non-negotiable.
BLOCK
CK-V007
Hash Integrity
The hash field must match the SHA-256 of the canonical JSON (with hash field excluded). Any tampering invalidates the manifest and triggers an audit event.
BLOCK
CK-V008
License Duration Bounds
Duration must be between 1 and 3,650 days (10 years max). Community tier capped at 365 days. Standard capped at 730. Premium negotiable up to 3,650.
BLOCK
CK-V009
Watermark Presence
watermark_hash must be a valid 64-character hex string (SHA-256). This ensures provenance tracking is embedded before synthesis begins.
BLOCK
CK-V010
Posthumous Policy Required
If the artist has Legacy Vault enabled, posthumous_policy and beneficiary_ids must be present. Warns if Legacy Vault is active but no beneficiaries designated.
WARN
CK-V011
Exclusivity-Tier Alignment
Exclusivity can only be true for Premium tier. Community and Standard licenses cannot be exclusive. Prevents low-tier exclusivity lock-ins.
BLOCK
CK-V012
Emergency Stop Override
If GABRIEL's emergency stop is active, ALL manifests are rejected regardless of validity. System-wide halt overrides individual consent.
BLOCK
Rights Ledger — Append-Only Record
Every consent event — grant, synthesis, distribution, revocation, rejection — is written to the Rights Ledger. The ledger is append-only: no record can be modified or deleted. Each entry includes a cryptographic hash that chains to the previous entry, creating a tamper-evident audit trail.
Ledger Entry Schema
Entry ID
Auto-incrementing, immutable
Event Type
consent_granted | synthesis_executed | license_activated | distribution_logged | revocation_triggered | synthesis_rejected | emergency_halt
Artist ID
Links to Identity Asset
Manifest Hash
SHA-256 of the consent manifest at time of event
Previous Hash
SHA-256 of previous ledger entry (chain integrity)
Timestamp
ISO 8601, UTC, millisecond precision
Actor
Who triggered the event: artist | licensee | system | gabriel
Metadata
JSON blob with event-specific data (amounts, rejection reasons, etc.)
Storage: D1 Table — rights_ledger
Database
noizyvox-consent (D1)
Write Mode
INSERT only. No UPDATE. No DELETE.
Chain Integrity
Each entry hashes the previous entry's hash + current data
Union Access
Read-only export API. Tamper-evident reports.
Dispute Window
Any ledger entry can be flagged for dispute within 90 days
Backup
Daily snapshot to KV (GABRIEL_KV). R2 cold archive when enabled.
