# Regulation Voices Product Spec

## Summary

Regulation Voices is a creator-owned voice library designed for predictable, low-surprise delivery.

It is intended for uses like:

- children's storytelling
- calm narration
- education
- accessibility reading
- select game and interactive contexts

It is not positioned as medical treatment.

## Product Thesis

Sensitive listeners do not need an unlimited wall of voices.

They need the right voice:

- stable pacing
- bounded dynamics
- softened harshness where appropriate
- clear licensing boundaries
- consent-first governance

## Landing Thesis

Regulation Voices should present itself as:

- artist-designed
- consent-locked
- predictable by design
- context-aware and ethically licensed

## Artist Flow

### RV-A0 Welcome

Core message:

`Publish a Regulation Voice. Voices designed for predictability, licensed with creator-first controls.`

### RV-A1 Consent And Allowed Use

Artists choose allowed contexts and disallowed contexts.

Required disallow examples:

- political persuasion
- adult content
- medical diagnosis claims
- implied endorsement

### RV-A2 Licensing And Payout

Minimum controls:

- indie tier
- studio tier
- enterprise tier
- usage model
- optional direct human-hire link

### RV-A3 Regulation Capture

Capture modes:

- calm
- gentle story
- focus
- sleep

The capture experience should avoid generic random text whenever possible.

### RV-A4 Profile Controls

Expose:

- sibilance softening
- prosody bounds
- speech rate
- breath texture
- mono default
- no-whisper variant

### RV-A5 Safety QA Gate

Pass conditions:

- no sharp transient spikes
- stable loudness
- no sudden pitch jumps beyond chosen bounds
- acceptable preview on speakers and headphones

### RV-A6 Publish Voice Card

A buyer should see:

- voice name
- best-fit contexts
- tone tags
- allowed use summary
- regulation modes
- link to creator profile

## Buyer Flow

### RV-B0 Casting Entry

Replace search-first browsing with intent-first casting.

### RV-B1 Project Intent

Collect:

- project type
- listener sensitivity
- desired voice feel
- required modes

### RV-B2 Curated Results

Return a small set of curated voices with:

- why this voice fits
- allowed contexts
- comfort constraints
- preview

### RV-B3 License Confirmation

Before generation, the buyer must confirm:

- allowed use
- disclosure requirements if applicable
- no implied endorsement

### RV-B4 Generation

Provide:

- calm, story, focus, sleep mode
- predictable output toggles
- usage receipt generation

## Settings Layer

### My Voices

Show:

- draft or published status
- contexts
- enabled modes
- quick pause action

### Licensing

Let artists edit context permissions and pricing without losing audit history.

### Safety And QA

Keep the regulation standard visible:

- predictable over dramatic
- stable over flashy
- clear over manipulative

### Usage And Royalties

Receipts should display:

- mode
- buyer tier
- date
- amount
- license route

### Revocation

Support:

- global pause
- context-specific pause
- buyer-specific pause where supported

Clarify that revocation blocks future generation and does not retract already exported content automatically.

## Marketplace Rules

- buyers license governed voice systems, not detached personality copies
- voice cards explain what the voice is for
- context enforcement happens at generation time
- receipts and audit trails are standard

## Launch Requirements

- creator verification
- regulation capture flow
- voice profile QA
- context-locked generation
- revocation controls
- transparent payout receipts
- security baseline aligned to local-first and authenticated operations

## Success Condition

Regulation Voices is working when creators feel protected, buyers feel clear on what they are licensing, and sensitive listeners encounter less surprise, not more.
