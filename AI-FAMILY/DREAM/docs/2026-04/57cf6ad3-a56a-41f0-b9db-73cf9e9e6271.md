# Cursorless Ramp: From Zero to Hot Rod

## The Mental Model First

Every Cursorless command is always just:

```
ACTION + TARGET
```

Target = **color** (optional) + **letter** (the hat character)
Example: `"chuck bat"` = delete the token with a hat on `b`

That's the whole game. Everything else is just expanding that vocabulary.

---

## Phase 1 — Week 1-2: Just the Hats

**Goal:** Stop using the mouse/keyboard for basic navigation and selection. Nothing else.

### Learn only these 4 commands:

| Say | Does |
|-----|------|
| `"take bat"` | Select token at `b` |
| `"pre bat"` | Move cursor before token at `b` |
| `"post bat"` | Move cursor after token at `b` |
| `"chuck bat"` | Delete token at `b` |

**The hat alphabet** — Cursorless uses NATO-style phonetics so letters are unambiguous:

```
air=a   bat=b   cap=c   drum=d   each=e   fine=f
gust=g  harp=h  sit=i   jury=j   crunch=k  look=l
made=m  near=n  odd=o   pit=p    quench=q  red=r
sun=s   trap=t  urge=u  vest=v   whale=w   plex=x
yank=y  zip=z
```

**Daily drill:** For one week, every time you reach for your mouse to click somewhere in code, say `"pre [letter]"` instead. That's it. Just navigation first.

---

## Phase 2 — Week 3-4: Colors + Copy/Paste

**Goal:** Add colors to disambiguate when multiple tokens start with the same letter. Add clipboard actions.

### Default hat colors:

```
default (no color word)
blue, green, red, pink, yellow
```

So if two tokens both start with `s`, you say `"blue sun"` vs `"red sun"`.

### Add these actions:

| Say | Does |
|-----|------|
| `"copy bat"` | Copy token |
| `"paste bat"` | Paste after token |
| `"carve bat"` | Cut token |
| `"bring bat"` | Copy token to cursor |
| `"move bat"` | Move token to cursor |

**Key insight:** `"bring"` is your new drag-and-drop. `"move"` is your new cut-paste. These two replace 80% of mouse-based code shuffling.

---

## Phase 3 — Week 5-6: Scopes (The Real Power)

**Goal:** Stop targeting individual tokens — start targeting *code structures*.

Scopes let you say what the thing **is**, not just where it **starts**.

### Core scopes to learn:

| Say | Targets |
|-----|---------|
| `"funk bat"` | The entire function containing/at `b` |
| `"class bat"` | The entire class |
| `"state bat"` | The statement |
| `"line bat"` | The whole line |
| `"block bat"` | Indented block |
| `"arg bat"` | Function argument |
| `"string bat"` | String literal |
| `"comment bat"` | Comment |

### Combos that are immediately useful:

```
"chuck line bat"     → delete entire line at b
"take funk bat"      → select entire function
"copy state bat"     → copy whole statement
"chuck arg bat"      → delete a function argument
```

**Use the visualizer** to learn scopes faster: say `"visualize funk"` and watch what lights up as you move around. Say `"visualize nothing"` to turn it off.

---

## Phase 4 — Week 7-8: Ranges + Lists

**Goal:** Operate on multiple things at once.

### Ranges (from → to):

```
"take air past bat"       → select from token a to token b
"chuck line air past bat" → delete lines from a to b
```

### Lists (this AND that):

```
"take air and bat"        → select both tokens (multi-cursor)
"chuck arg air and bat"   → delete two args at once
```

### Special targets:

```
"this"     → current selection
"that"     → last Cursorless target
"every funk"  → every function in the file
```

---

## Phase 5 — Advanced: Hot Rod Config

### 1. Customize your spoken forms

Edit `~/.talon/user/cursorless-settings/actions.csv`:

```csv
# Add aliases for common actions
bring|grab
chuck|nuke
```

Edit `scope_types.csv` to rename scopes to what feels natural to you:

```csv
funk|function|def
state|statement|stmt
```

### 2. Custom IDE actions (VSCode + Talon)

In `experimental/actions_custom.csv`, map spoken phrases to VSCode commands:

```csv
"format funk bat", "editor.action.formatDocument"
"rename bat", "editor.action.rename"
"extract bat", "editor.action.extractMethod"
```

### 3. Talon `.talon` file for your own shortcuts

Create `~/.talon/user/my_code.talon`:

```talon
# Quick refactor shortcuts
rename that:
    user.cursorless_command("rename", user.cursorless_target_mark("that"))
    
save file:
    key(cmd-s)

run tests:
    key(cmd-shift-p)
    insert("Run Test")
    key(enter)
```

### 4. Power combos to memorize last

```
"wrap bat with parens"       → surround with ()
"swap bat with cap"          → swap two tokens
"indent state bat"           → indent a statement
"dedent funk bat"            → dedent function
"next funk"                  → jump to next function
"every instance bat"         → select all occurrences
"extract bat"                → extract to variable (LSP)
"define bat"                 → go to definition
"reference bat"              → find all references
```

## One Rule to Keep It From Getting Overwhelming

> **Only add new commands when you feel friction.**

When you catch yourself reaching for the mouse or keyboard for something repetitive, that's your cue to find the Cursorless command for it. Don't try to learn the whole vocabulary at once — let pain points drive each phase forward.
