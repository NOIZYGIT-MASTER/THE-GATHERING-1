# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in THE-GATHERING or any NOIZY.AI system, please report it responsibly.

**Email:** security@noizy.ai (or rsplowman@icloud.com until email routing is configured)

**Do NOT:**
- Open a public GitHub issue for security vulnerabilities
- Share vulnerability details publicly before a fix is deployed
- Attempt to access other users' voice data or consent tokens

## Scope

The following systems are in scope:
- HEAVEN consent kernel (heaven.noizy.ai)
- NOIZY.AI website (noizy.ai)
- D1 databases (consent_log, gabriel_db)
- KV namespaces
- R2 voice DNA storage

## Response Timeline

- **Acknowledgment:** Within 48 hours
- **Assessment:** Within 7 days
- **Fix deployment:** Within 30 days for critical issues

## Human Voice Sovereignty

Voice data is sovereign. The 9 Never Clauses are constitutional and cannot be bypassed by any vulnerability. If you discover a way to circumvent consent checks, this is treated as a critical severity issue.

Thank you for helping keep voices safe.
