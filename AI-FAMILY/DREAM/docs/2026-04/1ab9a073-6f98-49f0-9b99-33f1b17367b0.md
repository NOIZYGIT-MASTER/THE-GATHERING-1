## iPhone → Pipeline: 3 Options

### Option 1: Airfoil (Pipeline Was Written For This)
The pipeline expects **raw 16kHz 16-bit mono PCM** on TCP 5555 — that's exactly what Airfoil streams.
1. Install **Airfoil** on Mac + **Airfoil Satellite** on iPhone (free)
2. In Airfoil: source = iPhone mic → stream to `GOD.local:5555`
3. `node dreamchamber-voice-pipeline.js` — done

### Option 2: Continuity Mic + sox bridge
iPhone appears as system audio input — but the pipeline has no system-mic mode, so bridge it:
```bash
sox -d -r 16000 -b 16 -c 1 -e signed-integer - | nc 127.0.0.1 5555
```
This captures system default mic at 16kHz mono and feeds the existing TCP server directly.

### Option 3: Audio Hijack
Record iPhone audio (via Continuity or Airfoil source), then on completion:
```bash
curl -X POST http://localhost:5556/webhook \
  -H 'Content-Type: application/json' \
  -d '{"path": "/path/to/recording.wav"}'
```
