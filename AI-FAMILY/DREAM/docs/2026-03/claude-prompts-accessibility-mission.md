## PROMPT 2: The AI Accessibility Layer Architecture
> Design the actual system.

```
I want you to architect an "AI Accessibility Layer" — a system that sits between disabled users and the technology/services they need to access. Think of it as a universal translator between human capability and digital systems.

Design this as a real technical architecture:

1. CORE ENGINE: An AI intermediary that can adapt its interface modality based on the user's abilities (voice, touch, eye-tracking, switch input, gesture, brain-computer interface)
2. MODALITY TRANSLATION: How it converts between what the user CAN do and what systems REQUIRE (e.g., a blind user needs to fill out a form designed for sighted users)
3. CONTEXT AWARENESS: How it learns each user's specific capabilities and preferences over time, without being invasive
4. SERVICE INTEGRATION: How it plugs into healthcare portals, government services, banking, transportation, emergency services
5. PRIVACY AND CONSENT: How medical/disability information is handled with explicit, revocable consent
6. OFFLINE CAPABILITY: Critical functions that must work without internet

I need system diagrams described in text, component definitions, data flow, and a clear separation between what exists today and what needs to be built. This should be something an engineering team could actually start from. Include ethical considerations at every architectural layer — this is Robert Stephen Plowman's initiative, and consent, dignity, and human agency are non-negotiable design requirements.
```
