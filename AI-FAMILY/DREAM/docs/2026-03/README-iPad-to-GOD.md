# MC96ECO — iPad → GOD Audio Bridge

## Architecture

```
  ┌─────────┐     USB-C      ┌──────────────────────┐     Output      ┌─────────┐
  │         │    Cable        │   M2 Ultra "GOD"     │    Device       │         │
  │  iPad   │ ──────────────▶ │   Audio Hijack 4     │ ──────────────▶ │ Speakers│
  │         │                 │   10.90.90.10         │                │ / DAC   │
  └─────────┘                 └──────────────────────┘                 └─────────┘
                                      │
                                      ▼ (optional)
                                ┌──────────┐
                                │ Recorder │
                                │ .aiff/.mp3│
                                └──────────┘
```

iPad audio flows over USB into Audio Hijack on GOD, where it can be monitored through speakers, processed with effects, and optionally recorded.

## Quick Start

### Step 1: Run the Setup Script

```bash
cd ~/path/to/audio-hijack-ipad-to-god
chmod +x ipad-to-god-setup.sh
./ipad-to-god-setup.sh
```

This will check for Audio Hijack, scan for your iPad, open Audio MIDI Setup if needed, and install the automation script.

### Step 2: Enable iPad as Audio Device (One Time)

1. Connect iPad to GOD via USB-C cable
2. Open **Audio MIDI Setup** (Applications → Utilities)
3. Find **iPad** in the left sidebar
4. Click **Enable**

After this, macOS treats the iPad as an audio input device.

### Step 3: Create the Session in Audio Hijack (One Time)

Audio Hijack sessions must be created in the GUI. This only needs to happen once.

1. Open Audio Hijack
2. Click **+ New Session** → Choose **Input Device** template
3. **Rename the session** to exactly: `iPad → GOD`
4. Click the **Input Device** block → Select **iPad** from the dropdown
5. Click the **Output Device** block → Select your preferred output (speakers, headphones, DAC)
6. **Optional:** Drag a **Recorder** block into the pipeline to capture audio
7. **Optional:** Drag a **Volume** block between input and output for level control

Your pipeline should look like:

```
[Input: iPad] → [Volume] → [Recorder] → [Output: Speakers]
```

### Step 4: Add Automation (Optional)

1. In the session, click the **Automations** tab
2. Add automation: **Event → Session Will Start** → **Action → Run Script**
3. Paste: `onSessionStart()`
4. Add automation: **Event → Session Did Stop** → **Action → Run Script**
5. Paste: `onSessionStop()`

### Step 5: Run It

Click the big **Run** button in Audio Hijack. iPad audio is now flowing to GOD.

## Script API Reference

The `iPad-to-GOD-AudioHijack.js` script provides these functions:

| Function | Action |
|----------|--------|
| `startRouting()` | Start the audio bridge |
| `stopRouting()` | Stop the audio bridge |
| `toggleRouting()` | Toggle on/off (default when script runs) |
| `getStatus()` | Report session status, blocks, and run time |
| `toggleMute()` | Mute/unmute output (session keeps running) |
| `toggleRecording()` | Enable/disable recorder block |

## AirPlay Alternative (No Cable)

If you prefer wireless routing instead of USB:

1. On GOD, go to **System Settings → General → AirDrop & Handoff**
2. Enable **AirPlay Receiver**
3. On iPad, swipe down for Control Center → **Screen Mirroring** or **AirPlay**
4. Select GOD as the AirPlay target
5. In Audio Hijack, use **Input Device → GOD's Microphone** (captures system audio)
   — OR use the **Application** source block targeting the AirPlay receiver process

Note: USB provides lower latency and higher fidelity than AirPlay.

## Troubleshooting

**iPad not appearing as audio device:**
- Ensure USB cable is connected (not just charging cables — use a data cable)
- Open Audio MIDI Setup and click Enable
- Try disconnecting and reconnecting
- Restart Audio Hijack

**No audio flowing:**
- Check that the session is running (green indicator)
- Verify iPad is playing audio
- Check volume levels in both Audio Hijack and iPad
- Ensure no blocks are disabled (muted)

**Latency issues:**
- Use USB instead of AirPlay
- Close unnecessary applications on GOD
- In Audio Hijack, reduce buffer size if available
