What We Anchor

Each day, we compute a cryptographic fingerprint of our audit log.

That fingerprint is published on public blockchains.

  • No audit data is published — only a hash (fingerprint)
  • The hash proves the audit log existed at that time
  • Any change to the log would produce a different hash

Where It's Anchored

Network What's Published How to Find It
Ethereum Hash in transaction input data Search tx hash on Etherscan
Bitcoin Hash in OP_RETURN output Search txid on Blockstream

Both are public and independently verifiable by anyone.

How to Verify (Step-by-Step)

1
Download the audit export

Request a compliance export for your date range from /operator/compliance/export

2
Compute the Merkle root locally

Use our verification script or compute it yourself:

cat audit_events.csv | cut -d',' -f1 | sort | sha256sum
3
Compare it to the published anchor

The computed hash should match the anchor record exactly.

4
Confirm blockchain timestamp

Verify that the blockchain transaction was mined before the audit period ended.

If all steps pass: The audit log has not been tampered with.

If any step fails: Something may have been altered — investigate further.

What This Does NOT Show

Verification is not surveillance.

  • No personal data exposed
  • No creative content revealed
  • No internal systems accessed
  • No private keys required

You verify integrity, not content.
