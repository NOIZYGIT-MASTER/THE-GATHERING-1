SECTION I
THE LEDGER DOCTRINE

Everything That Happened, Forever

The NOIZY LEDGER is not a database. It is not a log file. It is a legal record, a royalty engine, a compliance artifact, and a preservation document — all in one append-only, cryptographically anchored event stream. Every consent issued. Every voice synthesized. Every payout sent. Every revocation declared. Permanent. Ordered. Verifiable.

The Doctrine: Every action in the NOIZYVOX system produces exactly one Ledger event. Events are append-only — never updated, never deleted. Each event carries a hash of the previous event, forming a chain. The Ledger can answer any compliance, royalty, or provenance question about any voice, at any time, including 100 years from now.

SECTION II
IDENTITY ARCHITECTURE

The Five Identity Entities

Every Ledger event references at least one identity entity. Identity entities are permanent — they cannot be deleted, only revoked. The hierarchy flows from Actor → Voice → Descendant → Moment → Session.

ENTITY ID FORMAT DESCRIPTION SCOPE
ActorID RSP_001 / RSP_{NNN} The human performer. The sovereign. Permanent identifier assigned at enrollment. Never reused, never reassigned. Lifetime · Inheritable
VoiceID VID-{ActorID}-{ROLE}-{NNN} A specific voice identity under an Actor. An Actor may have multiple VoiceIDs (primary, character, character-2). DNA archive root. Per Actor · Multiplicity allowed
DescendantID DES-{VoiceID}-{NNN} A governed performance entity derived from a VoiceID. Has its own consent scope, revocation state, and royalty terms. Linked to parent VoiceID in provenance chain. Per VoiceID · Portfolio asset
MomentID MOM-{DescendantID}-{NNN} A single saved performance moment from a Playground session. Linked to EmotionTag, ConsentScope, and ConsentKeyID. Licensable asset unit. Per Descendant · Licensable
SessionID SES-{ActorID}-{timestamp}-{8hex} A single recording or Playground session. Container for one or more utterances. Linked to consent key active at session start. Per session · Temporary context
ConsentKeyID NOIZY-{ActorID}-CONSENT-{16hex}-AVA-{DOMAIN}-{JURISDICTION}-{DATE} The cryptographic consent anchor. Issued by consent-service, signed by HSM. Referenced in every Ledger event. Root of the provenance chain. Perpetual · Revocable · Auditable

SECTION IV
THE PROVENANCE SERVICE

The Proof That Travels With the File

The Provenance Service is the module that transforms internal Ledger records into externally verifiable proof. It is the bridge between NOIZY PROOF (internal authority) and C2PA (external verification). It must operate independently of the synthesis engine — it does not know which model ran, only what was authorized and what was produced.

The Provenance Separation Rule: The Provenance Service receives synthesis output events from voice-engine. It validates the Ledger entry. It generates the C2PA manifest. It signs it. It writes provenance.signed back to the Ledger. It does not import from voice-engine internals. It does not know model weights. It is model-agnostic by design.

SECTION VI
PREMIS PRESERVATION EVENTS

The Migration Contract

PREMIS is the international standard for preservation metadata that keeps digital objects usable over time. Every Voice DNA package and Descendant in NOIZYVOX generates PREMIS events. These events record not just what was created, but every transformation, migration, and format change — so the chain of custody is continuous across technological epochs.

PREMIS:CREATION → 2026-03-17
Voice DNA package created for RSP_001. Original capture: 44.1kHz PCM WAV, 24-bit. Significant properties extracted: spectral envelope, pitch contour, formant structure, emotional range baseline. SHA-256 fixity recorded. OAIS Submission Information Package assembled.
fixity: sha256:a1b2c3d4... | format: WAV_PCM_24 | model: MaskGCT-v2 (informational)
PREMIS:INGESTION → 2026-03-17
Package transferred to dna-archive. OAIS Archival Information Package created. Provenance Description Information attached. Redundant storage confirmed across 3 geographic regions. Designated community: RSP_001 + heirs + licensed voice engines.
archive_id: DNA-RSP001-001 | copies: 3 | storage: geo-redundant | checksum verified
PREMIS:DERIVATION → ongoing
Each Descendant created from this DNA package generates a derivation event. Descendant ID, parent ID, creation session, consent key, and audio fingerprint recorded. Provenance chain is traceable from any Descendant back to the original Voice DNA capture.
relationship: hasDerivedFrom → DNA-RSP001-001 | chain: continuous
PREMIS:MIGRATION → future (planned)
When SHA-256 is deprecated or current audio formats become obsolete: fixity verified against prior value, package converted to new format, new fixity computed, migration event recorded. Similarity score ≥ 0.90 required. Original package preserved alongside migrated package. Chain of custody unbroken.
trigger: algorithm_deprecation | similarity_threshold: 0.90 | original_preserved: true
PREMIS:RIGHTS_GRANTED → per license
Each licensing event generates a PREMIS Rights entity. License ID, licensee, use type, territory, term, royalty amount. Cross-referenced to consent key and Ledger event. The preservation record is also the royalty audit trail.
rights_basis: consent_key | royalty_floor: 85% | term: per_license

SECTION IX
THE 100-YEAR MIGRATION CONTRACT

Permanent vs. Replaceable

The Migration Contract defines exactly which components of the NOIZYVOX architecture must survive unchanged across 100 years, and which are expected and designed to change. Anything in the Permanent column is the trust rail. Anything in the Replaceable column is the creativity rail. This boundary is enforced at the interface level — Permanent services do not import Replaceable dependencies.

PERMANENT — TRUST RAIL
Never Changes (Only Migrates)
ActorID namespace and format
VoiceID lineage hierarchy
ConsentKeyID format and semantics
Never Clause NC_01–NC_10 definitions
85% royalty floor for RSP_001
Ledger append-only guarantee
Chain hash integrity
C2PA assertion schema (custom labels)
PREMIS event types
Voice Estate inheritance logic
Revocation semantics ("stop means stop")
REPLACEABLE — CREATIVITY RAIL
Expected to Change Every 18–36 Months
Synthesis model (MaskGCT → future)
Model weights and architecture
Audio codec and container format
Embedding space representation
RAG runtime / Director Brain logic
DreamChamber Studio UI framework
Cryptographic algorithms (SHA-256 → SHA-3)
C2PA signing certificate authority
Storage backend (S3 → future)
Payment rail (Stripe → future)
Voice quality benchmarks

The Migration Rule: Any item on the Replaceable list can be changed without touching the Permanent list. If a change to the Replaceable list requires modifying the Permanent list, that change is blocked until a migration event is recorded in the Ledger first. The migration event is the permanent record that a change occurred. The change is then permitted. The chain of custody is unbroken.
