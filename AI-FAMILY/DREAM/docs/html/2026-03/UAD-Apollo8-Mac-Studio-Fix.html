UAD Apollo 8 × Mac Studio M2 Ultra

Complete Fix Guide — Kernel Extension & Privacy/Security Resolution
🔍 What's Happening:
Your UAD Apollo 8 connects via Thunderbolt and requires a kernel extension (kext) to communicate with macOS. On Apple Silicon Macs (your M2 Ultra), Apple locks down kernel extensions by default for security. Even though you disabled SIP with csrutil disable, that alone is not enough on Apple Silicon. You need a different security change — specifically, the Reduced Security policy with kernel extension management enabled, done through the Startup Security Utility in Recovery Mode. That's why Universal Audio isn't appearing in Privacy & Security — macOS hasn't been told to even look for third-party kernel extensions yet.
⚠ Important: On Apple Silicon, csrutil disable is NOT the same as enabling kernel extensions. They are separate security mechanisms. You need to use the Startup Security Utility in Recovery Mode to lower the security policy. Also: re-enable SIP after this process — you don't need it disabled, and leaving it off introduces unnecessary risk.
1
Complete Clean Uninstall
Remove ALL existing UAD software and cached permissions
  1. Open Finder → Applications and look for UA Connect (or UA Uninstaller). If it has an uninstaller, run it first.
  2. Open Terminal and remove leftover UAD files. Run each line:
    sudo rm -rf /Library/Audio/Plug-Ins/HAL/UADDriver.plugin sudo rm -rf /Library/Audio/Plug-Ins/Components/UA* sudo rm -rf /Library/Audio/Plug-Ins/VST/UA* sudo rm -rf /Library/Audio/Plug-Ins/VST3/UA* sudo rm -rf /Library/Extensions/UAD* sudo rm -rf /Library/Extensions/Universal* sudo rm -rf "/Library/Application Support/Universal Audio" sudo rm -rf ~/Library/Preferences/com.uaudio.* sudo rm -rf ~/Library/Caches/com.uaudio.* sudo rm -rf "/Applications/UA Connect.app" sudo rm -rf "/Applications/Universal Audio"
  3. Clear the kernel extension staging area (this is the key step many guides miss):
    sudo kmutil clear-staging
  4. Verify no UAD extensions remain loaded:
    systemextensionsctl list
    If you see any Universal Audio entries, note them. They'll be cleared after the reboot process.
  5. Restart your Mac Studio. Don't skip this. A clean reboot clears the extension cache.
2
Re-Enable SIP + Set Reduced Security
Boot into Recovery Mode on the Mac Studio
💡 Tip: Open this guide on your iPad or phone before starting — you won't have a web browser in Recovery Mode.
  1. Shut down the Mac Studio completely. Not sleep — full shutdown.
    sudo shutdown -h now
  2. Press and hold the power button on the Mac Studio until you see "Loading startup options" on screen. Keep holding until the options appear (about 10 seconds).
  3. Click Options, then click Continue. You're now in Recovery Mode.
  4. If prompted, select your user account and enter your password.
  5. First: Re-enable SIP. From the menu bar, go to Utilities → Terminal and run:
    csrutil enable
    You should see: "System Integrity Protection is on."
  6. Close Terminal. Now from the menu bar, go to Utilities → Startup Security Utility.
  7. You'll see your startup disk (usually Macintosh HD). Click on it to select it.
  8. Click Security Policy…
  9. Select Reduced Security.
  10. This is the critical checkbox: Check the box that says:

    ☑ "Allow user management of kernel extensions from identified developers"

    This is the setting that was missing. Without this, macOS will never show third-party kernel extensions in Privacy & Security — no matter what you do with csrutil.
  11. Click OK. Enter your admin password if prompted.
  12. Click Apple menu → Restart from the menu bar.
Why this matters: On Intel Macs, csrutil disable was the nuclear option that opened everything up. On Apple Silicon, Apple separated the security layers. SIP controls system file protection. The Startup Security Utility controls what kinds of code can run at the kernel level. You need to address the second one for UAD.
3
Fresh UAD Software Install
Download and install LATEST version from Universal Audio
  1. After reboot, make sure your Apollo 8 is connected via Thunderbolt and powered on.
  2. Go to uaudio.com/downloads and download the latest version of UA Connect. Do NOT use an old installer — get the freshest one.
  3. Run the installer. It will ask you to restart when finished. Restart immediately.
  4. After restarting, you have a 30-minute window. This is critical.
    macOS only shows the "Allow" button for new kernel extensions for 30 minutes after installation. If you miss it, you have to uninstall and reinstall.
4
Approve the Kernel Extension
The part that wasn't working before — it should now
  1. Immediately after reboot, go to Apple Menu → System Settings → Privacy & Security.
  2. Scroll down to the Security section. You should now see a message like:

    "System software from developer 'Universal Audio' was blocked from loading."

    or

    "Some system software was blocked from loading."
  3. Click Allow. Enter your admin password.
  4. If it says software from multiple developers was blocked, a list will appear. Make sure to check/allow Universal Audio specifically.
  5. You'll also see a "Background Items Added" notification from UAD. Allow all of these. They are required for Apollo hardware to function.
  6. Restart your Mac Studio one more time.
If the "Allow" button still doesn't appear: This means Phase 2 didn't take. Go back to Recovery Mode and verify that Reduced Security is selected AND the kernel extension checkbox is checked. Both must be active.
5
Verify Apollo 8 Is Working
Confirm everything is talking to everything
  1. Open UA Connect. It should show your Apollo 8 as connected and the firmware as current.
  2. Open System Settings → Sound → Input. The Apollo 8 should now appear as an available input device.
  3. Open System Settings → Sound → Output. The Apollo 8 should also appear here.
  4. Open Audio MIDI Setup (in /Applications/Utilities). The Apollo 8 should appear in the device list with its full channel configuration.
  5. Terminal verification:
    system_profiler SPAudioDataType | grep -A 5 "Apollo"
    You should see your Apollo 8 listed with its manufacturer, sample rate, etc.
  6. If you have SwitchAudioSource installed:
    SwitchAudioSource -a -t input | grep -i "Apollo" SwitchAudioSource -a -t output | grep -i "Apollo"
  7. Open your DAW (Logic, Ableton, Luna, whatever you use) and set the Apollo 8 as the audio interface. Record a test signal. If audio flows, you're golden.
6
Nuclear Option (If All Else Fails)
The scorched-earth approach that has fixed the most stubborn cases
Only do this if Phases 1-5 didn't work. This is more aggressive and involves resetting the kernel extension cache at the Recovery Mode level.
  1. Uninstall all UAD software (repeat Phase 1).
  2. Boot into Recovery Mode (hold power button → Options → Continue).
  3. Open Terminal from Utilities → Terminal and run the Panic Medic reset:
    kmutil trigger-panic-medic --volume-root /Volumes/Macintosh\ HD
    (Adjust the volume name if yours is different.)

    This clears corrupted kernel extension caches that may be preventing the Allow button from appearing.
  4. While still in Recovery Mode, confirm Startup Security Utility has Reduced Security + kernel extension checkbox enabled (Phase 2, steps 6-11).
  5. Re-enable SIP if it's still disabled:
    csrutil enable
  6. Restart. Install fresh UAD software (Phase 3). Approve within 30 minutes (Phase 4).
Last resort: If even this doesn't work, check if Find My Mac is interfering. Some users have reported that turning off Find My (in System Settings → Apple ID → Find My) before the Recovery Mode process resolves persistent kernel extension approval failures. Re-enable it afterward.
