Real-Time Casting Engine

The core power. Instant, intelligent casting while the creator is actively working. AVA reads the content in real time, searches the Consent-Locked Vault, and surfaces real human performers before the creator has even finished thinking about what they need.

01
Content Analysis — Real Time
Script, audio, or scene is analysed as the creator works. AVA extracts emotional requirements, narrative tone, character archetype, language needs, and technical specs. The RAG pipeline grounds this in the full Vault context.
02
Voice Matching — Consent Vault Search
AVA searches the Consent-Locked Vault against emotional range tags, language flags, character archetypes, availability, and performance history. Never Clauses are checked before any result surfaces. No voice appears without a valid HMAC-SHA256 consent key.
03
Live Suggestions — Proactive Surfacing
AVA surfaces 3–5 real human performers. She does not wait to be asked. She says: "I have three perfect matches for this role. Do you want a more emotional delivery or more authoritative presence?" The creator makes a choice, not a search.
04
Instant Access — Live Mount, No Download
Voice is accessed via Mountain Duck live mount or RcloneView routed access. No file is duplicated. No download required. The voice is available inside the DAW or editor within seconds of the casting decision.
05
Execution — Local Synthesis + Quality Gate
Voice is synthesised locally on the M2 Ultra via Kokoro-82M using the actor's approved model. librosa validates output: 48kHz minimum, 16-bit minimum — the Never Clause floor. Voice is delivered directly into the active creative workflow.
06
Tracking — Usage Logged, Royalty Triggered
Every use is logged. Consent verified. Royalty event written to the SQLite Royalty Ledger: 75% to the actor, immutable, immediate. The Ferris wheel turns. The Memory System updates. AVA learns from this outcome for the next brief.

06 · Protocol

HVS Protocol Layer

The Human Voice Standard. Where NOIZYVOX stops being a platform and becomes a global protocol — the consent, identity, and quality infrastructure layer that any voice AI system can adopt. Think TCP/IP for voice identity. This is the standard the industry will need as regulation arrives.

The Protocol Vision
Every jurisdiction that regulates AI-generated voice will need a standard for consent verification, identity provenance, and quality certification. HVS is that standard — built from inside MC96ECO, proven in production, and designed to be licensable outward. VSI International. Major streaming platforms. Global advertising networks. All will need what HVS provides. We build it first. We define what the world adopts.

HVS Protocol Architecture

HVS-01
Identity Layer
Cryptographic actor identity anchored to HMAC-SHA256 consent key. Each voice has a unique, unforgeable HVS Identity Hash that travels with every licensed output. Verification is instant. Fraud is mathematically impossible.
HVS-02
Consent Layer
Scope-locked consent architecture. Each consent key encodes exactly what is permitted: usage type, duration, territorial rights, sublicensing restrictions. Revocation is immediate and propagates to all active licenses. The Never Clauses compile into every key.
HVS-03
Quality Certification Layer
Every output carries a Quality Certificate: 48kHz / 16-bit minimum verified, librosa-validated, governance hash signed. Any downstream system receiving an HVS-certified output knows it meets broadcast standard. No re-validation required.
HVS-04
Provenance Layer
Full audit trail from recording session to final licensed output. Every processing step logged. Every quality gate result recorded. Every use event timestamped. Provenance is immutable. This is what regulators will require. HVS has it from day one.
HVS-05
Economic Layer
75% perpetual royalty protocol embedded in the standard. The Royalty Ledger enforces it on every event. HVS-licensed outputs carry the economic terms inside the identity hash. No renegotiation. No expiry. Constitutional and computationally enforced.
HVS-06
Interoperability Layer
HVS is platform-agnostic. Any DAW, editor, streaming platform, or AI pipeline that implements the HVS SDK can verify consent, certify quality, and execute royalty distribution. This is the layer that turns NOIZYVOX from a product into an industry standard.
07 · Deal Engine

Autonomous Deal Engine

AVA negotiates licensing in real time. The creator sets their parameters once. AVA handles every deal within those parameters autonomously — without requiring the actor to be present, online, or aware in the moment. The royalty arrives. The wheel turned. They find out in the morning.

PHASE 01
Creator Sets Parameters
Actor defines their licensing floor: minimum project rate, usage types they accept, territories, exclusivity rules, and any client categories they will not work with. AVA stores this as their Deal Profile.
PHASE 02
Brief Arrives
Studio or client submits a casting brief via NOIZYVOX. AVA matches to the Vault and identifies performers whose Deal Profile aligns with the brief requirements and budget.
PHASE 03
AVA Negotiates
Within the actor's approved parameters, AVA negotiates terms with the client: pricing, scope, delivery timeline. She never exceeds the creator's stated limits. She optimises for the highest value deal the client will accept.
PHASE 04
Contract Auto-Generates
Collaboration Contract is created with agreed terms, 75% royalty split locked in, usage scope defined, HVS identity hash embedded. Contract is presented to both parties for digital signature.
PHASE 05
Actor Notified + Approves
Push notification: "AVA has negotiated a deal for you — review and approve." Actor reviews terms, approves or adjusts. Signature triggers the Vault to release the voice model for synthesis. No approval means no release.
PHASE 06
Execution + Royalty Flows
Voice is synthesised, quality-gated, delivered. Every subsequent use event triggers a royalty calculation — 75% to the actor, logged to the SQLite Ledger, trackable in real time. The wheel turns whether the creator is working or sleeping.
Why This Changes Everything
A voice actor on NOIZYVOX can be cast by a studio in Tokyo, have AVA negotiate the deal in real time, approve it from their phone in Montreal, and wake up the next morning to find the royalty has already hit their ledger — without a single human intermediary taking a cut of the 75% that belongs to them. That is not a feature. That is the Ferris wheel working exactly as designed.
